# AGR Framework Replication Dataset

<p align="center">
  <em>Replication Data Package for Automated GUI bug Report (AGR) Framework</em>
</p>

## Overview

This repository contains all intermediate and final data artifacts generated during the experimental pipeline of the **AGR framework**. 

All data files produced during execution are included to ensure transparency and reproducibility. Because the pipeline was executed incrementally, multiple versions of similar files (e.g., `_v1`, `_v2`) are retained to preserve the full experimental history and variations across different sampling rounds or pipeline iterations.

---

## 📁 Directory Structure & Included Files

To improve readability and usability, the data artifacts have been structurally organized into the following categories corresponding to their role in the AGR pipeline:

### 1. `raw_data/`
Contains the raw, unprocessed user feedback datasets directly collected from sources.
- `review_forum.jsonl`
- `support_forum.jsonl`
- `user_feedback.json`

### 2. `preprocessed_and_structured/`
Contains feedback data that has been cleaned, preprocessed, and formatted into structured schemas expected by the AGR pipeline.
- `old_structured_feedback.jsonl`
- `structured_feedback.jsonl`
- `structured_feedback_v2.jsonl`

### 3. `retrieved_evidence/`
Contains the retrieved external evidences and artifacts gathered during the information retrieval phase of the pipeline.
- `retrieved_feedbacks.jsonl`
- `new_retrieved_feedbacks.jsonl`
- `new_retrieved_feedbacks_v1.jsonl`
- `new_retrieved_feedbacks_v2.jsonl`

### 4. `generated_reports/`
Contains the final reports, including the bug reports generated by the agent.
- `generated_reports.jsonl`
- `generated_reports_v2.jsonl`

### 5. `stored_memory/`
Contains memory artifacts and intermediate context stored during the agent's execution.
- `stored_feedback.jsonl`
- `stored_feedback_v2.jsonl`

### 6. `evaluation_and_filtering/`
Contains artifacts used for evaluating the quality of feedback and filtering out useless reviews.
- `useless_review.json`
- `useless_review_v2.json`

### 7. `baselines_and_controls/`
Contains the output datasets from the control setups and baseline comparisons (e.g., without the agent or without retrieval).
- `control_without_agent_20b.xlsx`
- `control_without_retrival_20b.xlsx`

### 8. `auxiliary/`
Contains auxiliary resources and parallel corpora used to aid the process.
- `bugs_corpus.jsonl`
- `urls.txt`

## 🌐 Data Sources

All raw data originate from publicly available Mozilla open-source platforms:

- **Firefox Support Forum**  
  [https://support.mozilla.org/en-US/questions/firefox](https://support.mozilla.org/en-US/questions/firefox)
- **Firefox Official Support Documentation**  
  [https://support.mozilla.org/en-US/products/firefox](https://support.mozilla.org/en-US/products/firefox)
- **Mozilla Bugzilla** *(Firefox product components and historical bugs)*  
  [https://bugzilla.mozilla.org/describecomponents.cgi?product=Firefox](https://bugzilla.mozilla.org/describecomponents.cgi?product=Firefox)

These datasets are fully reproducible and can be independently re-collected from the official sources above.

---

## 📝 Notes on Reproducibility

- **Public Availability:** All upstream data are publicly accessible.
- **No Proprietary Data:** No proprietary or confidential datasets are used.
- **Execution Trace:** The file size differences between versions simply reflect different sampling rounds. This repository represents a complete and rigorous execution trace of the experimental workflow.
